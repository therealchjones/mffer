Welcome.

mffer is designed to be run as a web app on the Google Cloud Platform. To deploy the application as a project of your own, you must register it as an Oauth 2.0 client, add your own API key, and set up a Google Sheet for storage.

If you just want to use the web app, you don't need to do this: just visit mffer.org.

If you want to run, build, modify, or test the command-line mffer program, which is used for extracting data and reporting it in a way the web app can use, you may not need to do this: visit mffer on GitHub.

However, if you want to modify and test the web app or deploy a version yourself, then

Are Google Apps Script and Google Cloud Platform set up?

They're probably at least partly configured if you've gotten this far. For the webapp to work properly, you'll need to ensure those are complete by following the "Deploying the webapp" instructions in the mffer Development Guide. Once you've completed that, enter the identifiers from the "Credentials" section of your project below.

After submitting the below keys, you *may* be asked to sign into your Google account and allow appropriate access for the app. Check the permissions request carefully and ensure you're using the account that manages your Google Cloud Platform project.
